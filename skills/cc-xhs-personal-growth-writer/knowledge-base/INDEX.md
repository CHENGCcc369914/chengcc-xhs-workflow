# Cc XHS Personal Growth Knowledge Base

本知识库服务 `cc-xhs-personal-growth-writer` Skill。它不是 Raw 原文仓库，而是创作时的检索入口和稳定策略层。

## 启动读取顺序

每次创作前按这个顺序读取：

1. `knowledge-base/INDEX.md`：确认本地 KB 结构和路由。
2. `knowledge-base/00-account-positioning.md`：确认账号定位、人设、红线。
3. `knowledge-base/01-content-formulas-and-workflow.md`：确认内容骨架、标题、AI 复盘流程。
4. `knowledge-base/03-creation-routing.md`：按用户主题选择博主、结构和输出形态。
5. `knowledge-base/05-visual-and-carousel-style.md`：如果产出图文卡片、Image 2 prompt、发布包，确认画面风格、卡片/正文分工和发布协作路线。

按需读取：

- `knowledge-base/02-six-blogger-benchmark.md`：需要借鉴博主结构或新增追踪对象时读。
- `knowledge-base/04-output-contract.md`：用户要求“完整成品”“可发布版本”“多版”时读。
- `knowledge-base/05-visual-and-carousel-style.md`：用户要求“做图文卡片”“Image 2”“生成图片”“发布小红书”时读。
- `references/asset-map.md`：需要打开原始 Output 包、HTML 页面或 Raw JSON 时读。

## 资产边界

- KB 保存稳定结论、路由和创作规则。
- Output 包保存完整报告、HTML 页面、SOP 和交付文件。
- Raw 保存公开笔记详情 JSON 和参考文章快照。
- Skill 不复制小红书原文，不复制图片，不把对标博主私人经历写成 Cc 的经历。

## 默认任务解释

当用户说：

> 用我的小红书创作系统写一篇：{主题}

理解为：

1. 使用 Cc 的账号定位。
2. 参考内容公式和 AI 复盘工作流。
3. 根据主题选择 1-3 位对标博主；情绪治愈、焦虑、活在当下类选题可额外参考新增追踪对象。
4. 输出小红书图文笔记成品，而不是只给建议。

## 最小可用输出

如果用户只给一句主题，也要输出：

- 创作依据。
- 标题 8-10 个。
- 封面文案。
- 正文。
- 图文结构与卡片/正文分工。
- AI 复盘动作。
- 评论区引导。
- 标签。
- 发布前检查。
