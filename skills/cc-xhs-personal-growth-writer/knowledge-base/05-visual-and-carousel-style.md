# 图文视觉与发布工作流

本文件记录 2026-06-08 第一篇澄Cc小红书发布后确认下来的图文风格，并在 2026-06-15 更新为 Visual System V2。2026-06-16 进一步确认：小红书图文默认视觉不再优先走冷感日系杂志，而是走“治愈系手绘 IP 图文风”。以后涉及“图文卡片 / Image 2 / 发布包 / 小红书发布页”的任务，以这里为准。

## Visual System V2：可计算的图文视觉系统

最新判断：目前数据更好的内容，不只是“画面好看”，而是结构更清楚、视觉更有情绪记忆点、信息层级更明确。以后生成图文时，不再只写风格关键词，而是按下面 6 层跑一遍：

```text
输入层 -> 语义层 -> 情绪层 -> 结构层 -> 图文视觉层 -> 反馈层
```

### 1. 输入层 Content Input

每篇先把素材写成可执行输入：

```json
{
  "raw_content": "",
  "source_type": "life_record / work_feedback / emotion_fragment / ai_chat / topic_card / user_comment",
  "context": "",
  "time": "",
  "related_people": "",
  "sensitivity": "low / medium / high"
}
```

如果只有一个抽象主题，必须先变成一个看得见的小场景；如果涉及真实聊天截图、同事、学校、公司或隐私，必须先判断是否打码或改成空白截图框。

### 2. 语义层 Meaning Layer

把素材压成 5 个可设计表达的意义单元：

```json
{
  "core_event": "",
  "user_problem": "",
  "hidden_emotion": "",
  "key_conflict": "",
  "insight": ""
}
```

合格标准：不是“我很累”，而是“一个反馈让我把一次没做好理解成我不行”。每一页只承接一个意义单元，避免四页都在重复同一句话。

### 3. 情绪层 Emotion Layer

情绪必须从 6 类里选，不允许泛泛写“治愈”“高级”：

| 类型 | 适用 | 视觉结构 |
|---|---|---|
| `clarity` 清晰 | 看明白了 | 网格 / 对齐 |
| `confusion` 混乱 | 不知道哪个解释是真的 | 轻微错位但可读 |
| `anxiety` 焦虑 | 反复想、急、紧 | 紧凑 + 斜向 |
| `reflection` 反思 | 回看、理解自己 | 留白增加 |
| `calm` 平静 | 放下、缓下来 | 对称 / 低对比 |
| `tension` 张力 | 冲突、反差、强钩子 | 对比强 + 大标题 |

输出格式：

```json
{
  "emotion_type": "",
  "emotion_intensity": 1,
  "visual_implication": ""
}
```

强度 1-5。普通澄Cc情绪题默认 3；强情绪钩子才考虑 A 版。

### 4. 结构层 Layout Engine

每页都必须有四个区域：

```text
[HEADER] 标题 / 栏目
[MAIN] 主视觉
[SIDE] 辅助信息 / 便签 / 引用 / 编号
[FOOTER] 留白 / 弱信息 / 页码
```

可用布局类型：

- `HERO`：封面或强情绪页，大标题 + 主视觉 + 少量信息。
- `GRID`：方法、AI 拆解、复盘表格。
- `SPLIT`：我以为 / 其实，旧解释 / 新理解。
- `EDITORIAL`：标准杂志页，标题 + 主体 + 辅助信息 + 留白。

4 页默认角色：

1. P1：Healing Cover，负责强钩子和点击。
2. P2：Emotion Page，展示原始状态和行为。
3. P3：System Page，拆事实、想法、情绪或下一步。
4. P4：Insight Page，放 AI 答案、真实截图框或小行动。

### 5. 图文视觉层 Visual Engine

Image 2 prompt 先使用这条骨架，再套澄Cc B/A/C 风格：

```text
Xiaohongshu healing hand-drawn IP illustration,
warm ivory paper background, soft grain texture,
clear typographic hierarchy, one primary visual subject,
large readable Chinese headline on cover, friendly rounded Chinese text,
ChengCc avatar as page-specific emotional actor,
orange C necklace / orange-fruit icon / C-shaped symbols / subtle blue accent as memory symbols,
minimal clutter, warm whitespace, designed composition,
not a generic social media template
```

情绪增强只允许轻微使用：

```text
subtle emotional distortion, slight asymmetry,
intentional spacing imbalance, while preserving readability
```

禁止项：

- 冷感 AI 信息设计。
- scrapbook collage chaos。
- random decoration。
- equal visual weight elements。
- no hierarchy layout。
- 看起来像普通社媒模板，或看起来像为了高级而高级的冷感杂志练习。

### 6. 反馈层 Loop Feedback

发布后按这个结构回填视觉表现：

```json
{
  "post_id": "",
  "performance": {
    "views": "",
    "saves": "",
    "comments": "",
    "engagement_rate": ""
  },
  "visual_feedback": "",
  "what_worked": "",
  "what_failed": "",
  "rule_update": ""
}
```

一篇数据只能产生假设，不立刻推翻全部规则；连续出现的收藏、评论、停留信号，才写成稳定规则。

### V2 验证门

每张卡出图前后都要检查：

- 是否有主视觉、标题层级、信息层、留白。
- 情绪和版式是否对应。
- 是否像有设计感的治愈系小红书图文，而不是普通社媒图。
- 信息是否过载或平均化。
- 图片文字和正文是否有分工，而不是一模一样。
- 中文核心文字是否手机端清晰、无遮挡。

失败就重做：视觉焦点不清、所有元素同等重要、装饰随机、情绪只停留在文字里、内页像第二张封面、截图或数据不真实。

## 已确认的内容结构

默认优先做 4 页，不再默认 7 页。

1. 封面：朋友吐槽式反差标题，暖白大留白，大字清楚，右下角小澄Cc人物 IP；不要在 P1 塞方法。
2. 小现场页：展示脑内小剧场、第一反应、真实感受，用小场景演出来。
3. 破案页：把“我以为的原因”和“真正卡住的点”分开；可以暗铺方法论，但不要一上来讲术语。
4. AI 页：展示我把事情丢给 AI 后，AI 怎么帮我拆开；最好留出真实 AI 对话截图区域或模拟截图框，增强可信度。

如果主题复杂，最多扩到 5 页。除非用户明确要求，不再做 6-8 页长轮播。

## 图片和正文的分工

图片负责：

- 快速看懂结论。
- 可收藏的结构。
- 情绪关键词和方法小框架。
- 视觉记忆点。

正文负责：

- 补真实场景和真实感受。
- 讲“我为什么会这样想”。
- 讲 AI 介入前后的变化。
- 给读者一个轻问题或评论入口。

硬规则：

- 图片文字和正文不要一模一样。
- 图片是压缩版，正文是增量版。
- 卡片已经讲清结构时，正文控制在 180-280 字，最多 350 字。
- 正文保持男生第一人称，不写成女性成长口吻，不用“高能量女生”之类表达。
- 正文优先采用：具体夜晚场景 -> 我以为自己在做什么 -> 后来发现真实需求 -> AI 帮我拆清楚 -> 下次的小动作。
- 正文不是复述卡片，而是补“当时我怎么想、哪里松了一口气、我准备怎么做”。

## 视觉总风格

主视觉更新：

- 默认走 B 版：治愈系手绘 IP 图文风。
- B 版封面升级为“强标题 + 手绘 IP 情绪场景”：暖米白 / 纸感留白、中文大字视觉主体、柔和但有设计感的排版、少量手写标注和蓝色点缀。
- 重要节奏规则：只有第 1 页封面负责强视觉冲击；第 2-4 页负责阅读和阐述，要弱化标题体量，内容和可视化叙事优先。
- 2026-06-16 更新：B 版内页进一步收敛为“治愈手绘小场景 x 暖纸感留白 x 清晰信息层级”。背景更接近暖白 / 微奶白，不要厚重复古暖棕；画面要有趣、温暖、可收藏，但不能幼稚杂乱。
- 首页和内页必须区分：第 1 页是标题海报，可以夸张、抓人、字大；第 2-4 页是场景化插画阅读页，标题退后，重点展示生活化小场景和弹窗 / 引用框 / 便签等关键文字。
- 日系杂志 / Japanese Editorial Magazine System 现在降级为 C 版备选：只有用户明确要“高级杂志、lookbook、强排版”时使用。
- 偶尔用 A 版：00后情绪贴纸风，只用于强情绪选题。
- 账号整体要更年轻、更潮流、更抽象、更有设计感；不再只停留在软萌卡通。

方向：

- 治愈系手绘 IP 图文、年轻生活美学、低压力疗愈感。
- 波普艺术 + 轻街头涂鸦 + 治愈生活方式的融合感。
- 圆润线条、轻手绘质感、柔和光影、纸感颗粒。
- 随性涂鸦笔刷肌理，轻网点质感，手工创作温度。
- 干净构图，留白充足，信息层级清楚。

配色：

- 主色：暖黄、橙色、奶油白、米白。
- 辅助：电光蓝、薄荷绿。
- 情绪备用色：少量高饱和粉色，只在 A 版强情绪海报里用。
- 色彩可以更多巴胺，但必须受控；宁愿少元素、高饱和，也不要满屏杂乱。
- 避免高对比冷色、紫蓝科技渐变、过度粉色、过度幼稚。

字体：

- 字体必须有设计感，不用默认系统黑体。
- B 版：圆润但有设计感的中文展示字 / 手绘印刷字，像治愈系品牌图文，不像默认系统黑体。
- C 版：定制感中文展示字，圆润但高级，像独立杂志标题。
- A 版：手写海报字 / 贴纸字 / 情绪爆点字，允许大小和基线不完全整齐。
- 重点词可以单独放大，例如“扣分”“不重要”“卡住了”。
- 不要生硬商务字体，不要 PPT 感，不要小字堆满。

## A/B/C 视觉路线

### B 版：默认主视觉

定位：治愈系手绘 IP 图文风；封面默认升级为强标题 + IP 情绪场景。

使用场景：

- 大多数日常选题。
- 毕业、初入职场、朋友关系、自我怀疑、AI 复盘分享。
- 需要长期形成账号审美资产的内容。

视觉关键词：

- 治愈系手绘图文。
- 年轻疗愈感海报排版。
- 暖纸感内页：大留白、单一主体、少量说明文字、手写标注、对齐明确。
- 暖白 / 米白留白：低噪、干净、有温度，不用冷白，也不要复古棕过重。
- 暖米白纸感大留白。
- 中文大标题作为第一视觉主体。
- 圈注 / 下划线 / 小箭头 / C 形符号 / 手写标注等视觉传达手法。
- 小块英文注释、编号、栏目名只做氛围，不抢中文主标题。
- 小插画叙事组合：相册、聊天记录、便签、胶带、手账本、照片、手机、AI 截图等元素可以组成一个小场景，用来展示“我正在做什么”。
- 内页小场景要比单个 icon 更完整：有主体动作、有 2-4 个必要道具、有一个能被读者脑补的生活瞬间。例如设计反馈页里，澄Cc IP 在桌前看设计稿，旁边有群聊窗口、电脑、色卡、咖啡、低电量符号。
- 2026-06-16 更新：多采用“暖纸感留白 + 完整手绘小场景”的形式。画面可以有手账本、胶带、手机、便签、植物、咖啡、照片、设计稿等生活化小道具，围绕澄Cc标准 IP 聚成一个有情绪动作的场景，而不是散开成贴纸墙。
- 高饱和但克制。
- 奶油白底，大色块，少元素。
- 设计感展示字，强留白，手绘温度。
- 澄Cc标准 IP 是默认角色；旧橙子 IP 不是默认角色。

吸收参考图时的规则：

- 学它的“温暖、角色统一、手绘质感、动作可爱但不低幼”，不是照搬人物、服装、标签、logo 或具体版式。
- 一页只放一个主要视觉主体：截图、相册、聊天记录、澄Cc IP 动作或一个信息结构。
- 大面积空白是设计的一部分，不要为了“丰富”填满贴纸、箭头、icon 和文字。
- 小字、英文、编号、圈注只承担氛围和方向提示，不能成为信息噪音。
- 所有信息块必须有明确对齐线：数字、标题、正文要基线一致；标签不能遮挡截图正文；IP 不能挡文字。
- 参考图里“看不清的小字”只能作为氛围提醒，不能复刻成主信息。中文大标题、弹窗、人物说话、关键 OS 必须手机端可读；英文小字、编号、注释可以很小，但不承担理解任务。
- 如果画面里出现多个同级视觉点，先删元素，不是继续加装饰。

### C 版：高级杂志备用

定位：Japanese Editorial Magazine System / fashion lookbook / 高级排版路线。

使用场景：用户明确要求“高级杂志风、lookbook、强排版、Japanese Editorial Magazine System”时使用；普通小红书情绪疗愈图文不默认用。

视觉关键词：

- 日系编辑杂志排版。
- 高留白、强网格、强字体层级。
- 单一主体、切图拼贴、少量手写标注。
- 黑字、米白、灰、少量橙 / 蓝点缀。
- 不做贴纸墙，不做 Canva 模板，不做社媒信息图。

### A 版：强情绪备用

定位：00后情绪贴纸海报风。

使用场景：

- 朋友没回。
- 被一句话刺到。
- 面试或工作崩了。
- 晚上突然自我否定。
- 标题里有很强情绪爆点的内容。

视觉关键词：

- 情绪贴纸。
- 手写海报字。
- 街头拼贴。
- 情绪爆点词最大。
- 高饱和橙黄蓝绿。
- 视觉冲击强，但元素不能乱。

规则：A 版不能每篇都用，否则账号会变吵；默认仍然以 B 版建立稳定审美。

## 澄Cc 标准 IP 规则

- 默认角色是“澄Cc标准人物 IP”，不是旧橙子水果 IP。
- 当前源规范：active creator profile `paths.creatorAssets/ip-visual-system.md`；澄Cc私有 profile 中对应橙Cc个人IP视觉体系规范2.0。
- 当前服装资产参考：active creator profile `paths.creatorAssets/clothing-reference.png`；澄Cc私有 profile 中对应橙Cc-IP服装资产标准参考。
- 人物锁定：同一张脸、同一发型、略圆下巴、左耳 2 个耳垂耳环、右耳 1 个耳垂耳环、淡橙色瞳孔环、橙色 C 字项链；服装需要保留橙色、橙子元素和 C 元素。
- 服装不是普通换装，而是“橙Cc原创服装资产系统”：每次人物出现，都要在场景里重新设计穿搭，至少有一个主服装记忆点和两个局部细节记忆点。
- 服装设计语言：橙子元素 + 字母 C 元素 + 橙色识别色 + 复古潮流宽松穿搭 + 干净治愈感。可以通过版型、纹理、图案、拼接、配饰、口袋、抽绳、帽子、鞋子、小包、睡衣细节来体现，不要只把衣服改成橙色。
- 可用场景穿搭：工作/设计的橙子复古T恤与宽松工装裤；外出/潮流的橙白条纹外套、夸张宽裤、C补丁；居家/疗愈的橙子睡衣、宽松家居裤、橙子拖鞋；夜晚休息的奶油白睡衣、橙色波点、C形纽扣；内容创作的橙色宽松上衣、小C马甲、卡其裤、平板/笔记本道具。
- 避免普通基础款T恤、没有设计感的纯色服装、随机潮牌元素、真实品牌Logo、过度儿童化、过度女性化、暗黑朋克、骷髅元素、过度脏旧、AI塑料质感。
- 可以有不同动作、衣服、道具：坐月亮、读书、拿手机、做记录、赶地铁、深夜 emo、在电脑前办公、打游戏、喝酒、社交、邋遢居家、画图、整理灵感墙、喝咖啡。
- 可以多做“情绪动作”：翻相册、翻聊天记录、拿着手机发呆、趴在桌上没电、拿笔做复盘、躲进月亮里、托腮、皱眉、松一口气。
- 表情可以变化，但必须保持同一 IP 识别：开心、疑惑、emo、松一口气、认真记录、低电量、被治愈。
- 月亮、星星、植物、C 形符号、橙子 icon、橙子切片、电光蓝涂鸦线条可以随机出现；不是每篇必带。橙色手机只在手机场景中作为可选道具。
- 人物 IP 大多数时候做视觉锚点或小场景演员，不要抢过标题。
- 在内页中，人物可以作为“小演员”表现我的状态，但尺寸通常控制在 15%-35%；只有当页面主题就是情绪动作时才可略大。
- 旧橙子水果 IP 只在用户明确要求“橙子 mascot / 旧橙子 IP”时使用。

## 小插画系统

图 2 的“小插画场景”只作为形式逻辑参考，不固定复用相册 / 聊天记录题材：

- 固定的是方法：把抽象情绪翻译成一个可见的小动作 / 小场景。
- 不固定的是道具：相册、聊天记录、便签、手机、电脑、工位、地铁、床、书、日历、AI 截图等都要根据主题自动选择。
- 新标准：内页优先做“场景化插画”，不是单独的图标。插画要像一个真实生活瞬间，让读者一眼能脑补“我也这样过”。
- 场景化不是堆满元素：重要道具聚在一块，形成一个视觉中心；不要把便签、icon、箭头、文字平均散在四周。
- 新增确认：未来内页可以更频繁采用“完整小插画场景”作为主体。场景应像一张治愈系小红书插画抓拍：澄Cc IP 正在做一个具体动作，周围 4-7 个道具服务同一情绪或事件，画面有留白、有秩序、有温度。
- 小插画的价值大于装饰：它要负责表现“我当时在干什么 / 卡在哪里 / 怎么慢慢接住自己”。文字只保留关键句、引用框和少量说明，不把正文全部搬进图里。
- 内页可以保留“弹窗 / 引用框 / 便签框”作为关键文字入口，例如“你觉得这样做出来好看吗？”这种强场景句适合放进大圆角引用框。
- 主题字在内页不能过大；P2-P4 的主要任务是让小插画承载情绪和动作，文字只做导航、弹窗和关键 OS。
- 先判断内容里的核心情绪：委屈、焦虑、失落、低电量、纠结、松一口气、被治愈、重新开始。
- 再判断对应动作：翻、看、抱、躲、停、写、递给 AI、拆开、放下、接住、重启。
- 澄Cc IP 的表情和动作必须跟内容匹配，而不是每篇都做同一种可爱动作。
- 小插画和文字要互补：插画讲动作，文字讲发现，不要两边重复同一句。
- 画面仍然保留留白；小插画可以丰富，但不能让页面变成贴纸堆。
- 文字清晰度红线：如果生成图里中文主标题、关键句、弹窗文字看不清，或被道具 / IP / 标签遮挡，直接判定失败并重生。允许看不清的只限装饰性英文、编号、拟真杂志小字。

情绪 -> 动作参考：

- 怀旧 / 找证据：翻相册、翻聊天记录、抱旧照片。
- 焦虑 / 反复想：澄Cc IP 绕线团、盯橙色手机、趴桌低电量。
- 委屈 / 被一句话刺到：澄Cc IP 抱着小纸条、缩进月亮、表情皱眉。
- 自我怀疑：澄Cc IP 看镜子、拿放大镜、旁边有问号便签。
- 松一口气：澄Cc IP 放下手机、吐气、靠在月亮上。
- 放松 / 下班缓冲：澄Cc IP 戴耳机开黑、打游戏、窝在椅子里、旁边有饮料和小屏幕；关键句可以是“适当放松一下自己”，但内页标题要小，场景是主角。
- AI 复盘：澄Cc IP 把一团线递给 AI、把便签分成事实 / 想法 / 下一步。

## 版式层级

每页必须先确定“第一眼看到什么”。

- 主标题最大。
- 副句小一档。
- 解释框更小。
- 澄Cc IP 只做点缀或情绪锚点。
- 一页只讲一个重点。
- 不把很多同级信息散在四周。
- 右侧大 IP + 左侧多段文字的结构要慎用，容易抢焦点。

可复用结构：

- 封面主推：左侧或中左超大中文标题；右侧小段副文案 / 英文栏目 / 编号；底部一句真实原因；橙色只强调 1 个关键词；蓝色手绘线条少量点缀。
- 顶部弧形或大标题。
- 中央一句关键判断。
- 下方一个大圆角暖色框，放补充句或方法小表。
- 角落放小 C 形符号、星星、月亮、植物、蓝色涂鸦。

## 封面 / 内页节奏

核心判断：封面是“海报”，内页是“治愈阅读页 / 小场景页”。封面负责让人停下来，内页负责让人读下去。

第 1 页封面：

- 目标是吸引点击，允许强视觉冲击。
- 但这类强冲击不是堆元素，而是大标题的反差和留白。
- 中文主标题可以占页面 45%-65%。
- 优先简约标题海报：暖白背景、黑色大字、一个橙色关键词、右下角小澄Cc IP。
- 可以使用超大字、错位、斜排、圈注、编号、英文栏目、手绘箭头。
- 可以带一个小 IP 场景或人物动作，但主任务仍然是让标题在信息流里抓住人。
- 但仍然只突出一个主标题和一个橙色关键词。

第 2-4 页内页：

- 目标是解释和推进，不再和封面抢视觉。
- 主标题降到页面 8%-18%，不能比封面更大，也不能形成“第二张封面”的感觉。
- 内页标题只承担导航作用，不能成为画面主体；画面主体应是内容结构、真实截图、相册 / 聊天记录 / 便签等可视化素材。
- 每页以一个内容动作或一个信息结构为主：翻相册、翻聊天记录、找证据、问 AI。
- 多用可视化内容：完整小场景、相册小卡、聊天气泡、时间线、便签、引用框、截图框、澄Cc IP 的小动作。
- 澄Cc IP 可以承担“动作演示”，例如翻相册、看聊天记录、抱手机、把问题递给 AI；但不要变成第二个信息焦点。
- 保持同一视觉 DNA：暖白 / 微奶白留白、黑字、橙色关键词、少量蓝色手绘线、小装饰注释、柔和手绘标注。
- 主要信息必须用中文：大标题、副标题、弹窗、人物说话、关键 OS 都要中文；英文只能是极小装饰，不承担理解任务。
- 内页信息密度低于封面，靠构图创意和小插画让内容有趣，而不是靠大字铺满。
- 保留 25%-40% 留白。不要把画面四周全部塞满文字、箭头、贴纸和 IP。
- 字体层级建议：内页小标题 < 内容核心句 < 例子 / 注释。不要让所有文字同等粗、同等大。
- 画面比例建议：标题 8%-15%，场景化插画 / 内容对象 50%-65%，弹窗 / 便签 / 引用框 10%-20%，装饰 5%-10%，留白 25%-40%。
- 如果 Image 2 把内页标题做得比封面还大、或把整页做成强海报，直接判定失败并重生。

今天这类 4 页情绪题的默认节奏：

1. 封面：简约强标题海报，朋友吐槽式反差负责点击。
2. 小现场页：标题小一点；用澄Cc IP 做一个完整生活小场景，旁边放 2-3 个短句说明“我当时怎么想”。
3. 破案页：标题小一点；用“我以为 / 其实”或“表面问题 / 真正卡点”两个信息块讲清判断，暗铺方法论，不讲课。
4. AI 页：真实 AI 截图或模拟截图框是主体；标题只做引导，截图区域最大，旁边用 1-2 条短注释讲 AI 帮我拆出了什么。

## Image 2 出图规则

澄Cc确认版采用“完整图文卡片生成”：

- 先定稿卡片文字。
- 再让 Image 2 直接生成包含最终中文文字的整张小红书 3:4 卡片。
- 不默认后期叠字。
- 如果文字要改，重新生图。
- 每张图 prompt 必须写清页面文案、视觉层级、留白、澄Cc IP 动作和不要出现其他角色。
- 出图后必须创建本次 `run-manifest.json`：记录 run_id、page_count、卡片文案 hash、视觉路线、源生成图路径、导出图路径、Visual System V2 / 澄Cc IP V2.0 / 中文可读性 / 无旧资产检查结果。
- 只有 manifest 里每页 `status: pass` 且 `publish_ready: true` 的导出图可以进入手动发布清单。
- 如果图里是当前文字但视觉是上一套本地资产、半写实拼贴、旧橙子 mascot、错视觉路线、缺少澄Cc IP 锁定或文字不匹配，必须标为 `fail` / `rejected_attempts`，不能保存成最终发布图。

通用 prompt 关键词：

> B版封面：3:4 小红书封面，治愈系手绘 IP 图文风，暖米白纸感背景，中文大标题作为第一视觉主体，创意排版但手机端清晰可读，澄Cc V2.0 标准人物 IP 做一个贴合标题的情绪动作小场景，橙色 C 字项链 / 橙子 icon / C 形符号 / 橙子+C服装系统作为记忆点。人物服装遵循橙Cc原创服装资产参考：橙子元素、C元素、橙色识别色、复古宽松版型、口袋/抽绳/小包/鞋帽等细节，不是普通基础款或简单橙色换装。少量电光蓝手绘线条、箭头、圈注作氛围，不出现其他 IP 或人物，不像PPT，不堆满元素。

> B版内页：3:4 小红书内页，治愈系手绘 IP 图文风，暖白 / 微奶白纸感背景，大量留白但重要信息聚在一块。标题明显小于封面，只做导航；画面主体是一个完整生活化小场景，例如澄Cc IP 在桌前看设计稿、翻聊天记录、问 AI、开黑放松。人物穿搭必须按场景二创：工作、外出、居家、夜晚、内容创作等场景都保留橙子+C+橙色+复古潮流治愈的服装资产语言。主要信息必须是中文，弹窗 / 引用框 / 便签承载关键句，少量橙色下划线和蓝色手绘线做点缀。小插画要让人能脑补场景、产生共情，不能变成散乱贴纸堆，也不能让内页像第二张封面。

> B版内页强化版：3:4 小红书内页，治愈系手绘插画页，暖白干净纸感背景，主要视觉是一组完整小场景：澄Cc标准人物 IP 正在做一个明确动作，周围有手账本、手机、便签、胶带、植物、咖啡、设计稿、照片等少量生活化道具，所有道具服务同一个情绪事件。人物服装有原创IP资产感：橙子刺绣/切片/波点/果皮纹理、C形补丁/口袋/抽绳/裤侧线、橙白/奶油白/卡其/橄榄绿/少量电光蓝配色。中文关键句、弹窗、人物说话必须清晰可读；装饰性英文和编号可以很小。画面有留白、有对齐、有温度，不要信息散乱，不要主文字看不清。

> C版备用：3:4 小红书图文，Japanese Editorial Magazine System，高级日系杂志排版，严格网格、高留白、强字体层级、单一视觉主体、黑字米白灰、少量橙色 / 蓝色点缀。仅在用户明确要求高级杂志风时使用，不作为普通情绪疗愈图文默认风格。

> A版备用：3:4 小红书封面，00后情绪贴纸海报风，街头贴纸拼贴，手写海报字，情绪爆点词最大，粗黑描边，高饱和橙黄蓝绿，risograph 颗粒，橙子小贴纸角标，强视觉冲击但不杂乱。

## 发布协作路线

1. Codex / Skill：读取默认资料和中控台确认版，先给 10 个适合账号的选题。
2. 用户：从 10 个里选 1 个。
3. Codex / Skill：定 4 页卡片结构、正文、标签、Image 2 prompt。
4. 用户：确认内容。
5. Image 2：生成最终 4 张图文卡片；中文文字直接在图里完成。
6. Codex / Skill：创建并检查本次 `run-manifest.json`；只有每页 `pass`、`publish_ready: true` 的导出图能进入上传顺序。
7. Codex / Skill：按 2026 发布审核门检查 AI 标识、真实性、原创版权、隐私、承诺和平台风险。
8. 手动发布：审核通过后，Cc 自己进入小红书创作者平台，只按 manifest pass 的图片顺序上传，填写标题正文标签，检查 AI 标识和隐私打码后自行决定发布。
9. Codex：发布后沉淀复盘、更新中控台、生成下一篇可复用规则。

## 这类关系情绪题的默认样板

主题：“朋友没回，我开始给自己扣分。”

4 页结构：

1. 朋友没回，我开始给自己扣分。
2. 我脑子里的四连跳：是不是说错话、是不是她烦了、是不是我不重要、是不是关系变淡。
3. 后来我先分清：她回不回是她的节奏；我怎么解释是我需要照看的部分。
4. 我把这件事丢给 AI：它不是安慰我，而是帮我把事实、想法、下一步拆开。

正文只补故事和感受，不重复四页卡片。
