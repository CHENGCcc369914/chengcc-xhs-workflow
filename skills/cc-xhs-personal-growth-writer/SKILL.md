# Cc 小红书个人成长与情绪创作系统

目标：把 Cc 的账号定位、核心对标博主蒸馏、新增追踪对象、内容公式、AI 复盘写作组合成一个可反复调用的文案引擎。它现在默认作为 L4.5 runtime / single-post workflow 的写作子模块；只有用户明确要求轻量写作、不建 run、不跑流程时，才独立输出小红书文字层内容。

## 0. 适用范围

使用本 Skill 独立处理：

- 明确轻量的小红书文字稿、改写、扩写、标题池、封面文案、评论区引导。
- 个人成长、情绪、职场反思、关系边界、AI 复盘、经典方法论重读。
- 将一个小道理拆成多篇选题。
- 从用户的真实经历或模拟案例生成笔记。
- 结合对标博主结构做仿写，但不能复制原文。

如果用户只是说「做一篇澄Cc小红书」「用澄Cc默认资料写一篇」「帮我写一篇澄Cc小红书」「做一个澄Cc选题」，默认交给 `chengcc-content-loop-runtime` 建 run。不要因为用户用了“写一篇”三个字就把它降级成一次性文案。

如果任务涉及实际浏览、搜索、发布、评论或小红书账号后台操作，由 `xiaohongshu-ops` 作为平台执行器；浏览器流程遵守 `xiaohongshu-ops` 的 `openclaw` 规则。纯写作任务不需要打开小红书。

与小红书四件套协作时按下面路由：

- 对标博主最新动态、watchlist、note-card、RAG brief：先用 `xhs-blogger-intelligence`。
- 写作前需要参考低粉高爆案例、类似选题表达、爆文正文结构或 RAG 检索时：先由 `xhs-blogger-intelligence` 从当前 runtime `Creator Profile` 的 `ragCaseLibrary` 生成 compact RAG brief，再由本 Skill 写作。只有在 Cc 本机私有安装且没有 runtime profile 时，才使用 Cc 的本地案例库 fallback。
- 如果本 Skill 是在 `chengcc-content-loop-runtime` 的 run 内被调用，先读取当前 run 的 `01-bootstrap-context.md`。若其中有 `## Creator Profile`，优先使用该 profile 的 creator、positioning、voice、visual system、RAG case library 和 asset paths；不要静默回落到 Cc/澄Cc默认资料。
- 默认澄Cc单篇生产、完整工作流、跨篇记忆、发布包、图片、检测、复盘：`chengcc-content-loop-runtime` 主导。
- runtime 里的单篇发布包、10 个选题、4 页图文、Image 2、2026 发布审核、手动发布清单：`chengcc-xhs-workflow` 主导，本 Skill 只提供账号声音和内容写作。
- 明确只写标题、正文、封面文案、评论区引导、单篇轻量草稿：本 Skill 主导。
- 真实小红书浏览、上传、填发布页、评论回复：`xiaohongshu-ops` 主导平台动作。

本 Skill 不生成最终图片、不复制图片到发布目录、不创建手动发布清单、不判断平台后台数据。遇到这些任务时，把文案交给 `chengcc-content-loop-runtime` / `chengcc-xhs-workflow` 或 `xiaohongshu-ops`。

## 1. 必读知识库

开始创作前先读 `knowledge-base/INDEX.md`，再按任务读取知识库卡片和资产。

默认每次至少读取：

1. `knowledge-base/INDEX.md`
2. `knowledge-base/00-account-positioning.md`
3. `knowledge-base/01-content-formulas-and-workflow.md`
4. `knowledge-base/03-creation-routing.md`

按需读取：

- 要借鉴博主：读 `knowledge-base/02-six-blogger-benchmark.md`，必要时再读对应 HTML 页。
- 要输出完整可发布版本或多版：读 `knowledge-base/04-output-contract.md`。
- 要打开完整 Output、HTML 报告或 Raw JSON：读 `references/asset-map.md` 后按路径读取。
- 用户说「澄Cc默认资料」「参考我之前定好的资料」「参考我的自媒体操作台」「按我上传的结构」时：读 `references/asset-map.md`，再读取其中登记的澄Cc自媒体操作台 HTML；若该文件缺失，明确说明并继续使用 Skill KB + 运营辅助包 + 核心博主沉淀。
- 要做深度结构、方法论对照、长期定位：读取 Output 包中的 Biji 方法论对照与 V2 深度蒸馏。

不要声称“已参考某文件”，除非本轮确实读取过。

## 2. Cc 的账号底盘

默认账号定位：

> 个人成长与情绪 x 00后应届生 / 初入职场新手 x 朋友吐槽式破案 x AI 复盘分享。

一句话表达：

> 我用自己的毕业、关系和初入职场经历，记录一个 00 后新手如何把困惑慢慢看清楚；AI 只是我偶尔会分享的小方法。

核心内容结构：

1. 朋友会想听的小现场：真实或模拟的职场、情绪、关系、自我怀疑场景。
2. 错误解释：我当时以为是 A，脑子里怎么演。
3. 反差破案：后来发现真正卡住的是 B。
4. AI 复盘：让 AI 帮我拆事实、解释、情绪、下一步。
5. 小动作：读者今天就能偷走的一步。
6. 评论入口：留一个选择、站队、补充经历或“你是哪一种”的话口。

写作原则：

- 一次只讲一个小道理。
- 同一个知识点拆到不同场景反复写。
- 先完成，再完美。
- 第一人称，低姿态，不装导师。
- 默认像朋友吐槽和一起破案，不像老师讲课。
- 标题可以锋利，正文要温柔。
- AI 是复盘辅助，不是神谕。

红线：

- 不做心理诊断、疗愈承诺、职业收益承诺。
- 不照搬对标账号原文、私人经历或标题。
- 不用“绝对变好”“一定成功”等确定性承诺。
- 不把 AI 输出当最终真理。

## 3. 对标博主选择

根据选题选择 1-3 个主要参考对象：

- 情绪、内耗、心理动作：优先 `心理师 bestta`、`KnowYourself`。
- 主体性、能量回收、自我价值：优先 `芝士Rebecca.`；只学结构，不套女性成长身份。
- 职场主体性、强观点、行动闭环：优先 `小马赛克mosaic`。
- AI + 职场进阶、求职、产品路径：优先 `莉至`。
- AI 工具场景、营销、商业化表达：优先 `几口酱聊AI`。
- 需要机构级心理概念拆解：优先 `KnowYourself`。
- 年轻人焦虑、治愈自己、活在当下、低负担快乐收藏：新增追踪 `小柴人不纠结`；只学选题入口、情绪价值和收藏型图文节奏，不照搬 IP、标题和原文。

使用方式：

- 学结构，不学原话。
- 学钩子，不照搬标题。
- 学情绪递进，不复制经历。
- 学结尾互动机制，不复制评论话术。

## 4. 单篇创作流程

当用户只给一个主题，例如“我想写课题分离和职场内耗”，按这个流程执行：

1. 读取必读知识库。
2. 如果主题适合小红书案例对照，先通过 `xhs-blogger-intelligence` 或一个 bounded local retrieval 生成 3-8 篇类似低粉高爆案例的 compact RAG brief，提取标题钩子、真实铺垫、AI 转折问题、方法清单和收藏 CTA；不要直接复制原文段落。
3. 判断主题所属场景：情绪 / 职场 / 关系 / AI / 经典方法论 / 个人品牌。
4. 选择 1-3 位对标博主，先读博主调用卡，必要时读取对应蒸馏页。
5. 把主题压成一个小道理，不写大而全议题。
6. 构造一个真实或模拟的个人场景。
7. 生成 AI 复盘表：事实 / 解释 / 情绪 / 下一步。
8. 输出小红书文字层成品：标题、封面文案、正文、互动问题、标签。
9. 做风险检查：诊断承诺、成功学口号、空泛说理、AI 装饰化、照搬风险。
10. 如有新模式或用户反馈，沉淀到小红书知识库或输出包中，先说明写入范围。

## 6. 内容质量标准

一篇合格笔记必须满足：

- 有一个明确的小道理。
- 有具体场景，不只讲抽象道理。
- 有一个反差或误解：我以为 A，结果 B。
- 标题像朋友吐槽/破案线索，不像课程标题。
- 有“我以前也这样”的入口。
- 有经典方法论支撑。
- 有 AI 复盘动作，而且这个动作真的解决文中的问题。
- 有一个低成本行动。
- 有评论区问题。
- 语言像小红书笔记，不像报告。

## 7. 用户可用触发语

用户可以直接说：

```text
用澄Cc默认资料写一篇：{主题}
```

这句是最短触发语。默认含义：

1. 启用本 Skill。
2. 读取本地 KB：账号定位、内容公式、创作路由、输出契约。
3. 读取澄Cc自媒体操作台 HTML，遵守“真实小场景 -> 脑内小剧场/卡住的地方 -> 换个角度重新看 -> AI 整理一个下一步”。
4. 按主题选择 1-3 位对标博主，只学结构，不复制原文。
5. 输出可发布的小红书图文/正文包，并做风险检查。

也可以说：

```text
用我的小红书创作系统写一篇：{主题}
```

也可以说：

```text
这个选题帮我跑 Cc 小红书 Skill：{主题/场景/一句话经历}
```

如果信息不足，不要先问一堆问题。先基于默认账号定位写一个可用初稿，并在末尾列出“如果补充真实经历，可以增强的 3 个点”。
