## 核心资产目录

运营辅助包：

Cc 私有安装中的 Obsidian / Output 资料包。分享给朋友时，不要求朋友拥有该路径；朋友应在自己的 `Creator Profile` 中替换为自己的资料包或留空。

澄Cc自媒体操作台：

Cc 私有本地 HTML 中控台。模板运行时优先读取 active creator profile；朋友没有该文件时，不应阻断流程。

用途：当用户说「澄Cc默认资料」「参考我之前定好的资料」「参考我的自媒体操作台」「按我上传的结构」时读取。重点遵守其中的定位和写作结构：00后应届生 / 初入职场新手 -> 朋友吐槽式小现场 -> 我以为 A 其实 B 的反差 -> 脑内小剧场 / 卡住的地方 -> AI 只是帮我拆线团的小方法。

图文视觉确认版：

当前视觉真相源是 `../knowledge-base/05-visual-and-carousel-style.md`。本节只做路由摘要，不作为独立视觉规范使用。

- 4 页优先：封面 -> 真实感受 -> 暗铺方法论 -> AI 小答案。
- 图片讲结构和结论，正文讲真实感受，不重复。
- 默认走 B 版：治愈系手绘 IP 图文风。封面是强标题 + 澄Cc V2.0 标准人物 IP 情绪场景；内页是暖纸感小场景 + 清晰信息层级。
- 澄Cc标准人物 IP 是默认角色；旧橙子水果 mascot 只在用户明确要求旧路线时使用。
- 人物必须遵守 active creator profile `paths.creatorAssets/ip-visual-system.md`；澄Cc私有 profile 中对应橙Cc个人IP视觉体系规范2.0：同一脸型、黑色四六侧分、左耳 2 个耳垂耳环、右耳 1 个耳垂耳环、淡橙色瞳孔环、橙色 C 字项链。
- 服装必须遵守 active creator profile `paths.creatorAssets/clothing-reference.png` 的原创服装资产密度；澄Cc私有 profile 中对应橙子元素 + C 元素 + 橙色识别色 + 复古宽松治愈穿搭；不能退回普通基础款、半写实拼贴、随机潮牌或简单橙色换装。
- Image 2 直接生成带最终中文文字的完整图文卡片；如果改文字就重生图。
- 生成后的图片必须通过当前主题文字、Visual System V2、澄Cc IP V2.0、导出 run manifest 四项检查后，才能写入手动发布清单。
- 发布页标题正文由 Cc 手动上传和粘贴；Codex 负责内容、图片、审核清单和规则沉淀。

## 默认成品结构

一篇笔记的最低可发布形态：

1. 标题 8-10 个。
2. 封面文案 1 组。
3. 正文 1 篇。
4. AI 复盘 Prompt 或操作表。
5. 评论区问题 1-3 个。
6. 标签 5-8 个。
7. 发布前风险检查。

## 引用边界

- 不复制小红书原文和图片。
- 不把对标账号的私人经历写成 Cc 的经历。
- 输出中如需说明来源，用“参考了某类结构/表达机制”，不要写成“某博主说过”。
- Raw 是证据层，Output 是可执行资产层；优先从 Output 创作，只有需要核验时才读 Raw。
